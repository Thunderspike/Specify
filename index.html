Data Configuration Editor

The Data Configuration Editor currently acceps 3 top level keys:

{ "editable": boolean, "sortable": boolean, cells: { ... }}

The top level "editable" key can is setting every cell to be editable. The apps internal engine determines what "type" the cells will be

"sortable" on the other hand toggles the abillity to sort the rows based on the selected column. Click on a column in order to see this key in action

"cells" expects an object each key is a pointer to the cell in question - in the pattern "cYrX" where "cY" represents the column number, and "rX" represents the row number. X and Y are 0 based indices

"r0c0": { "type": "text" }, "r1c1": { "type": [ "flying", "lost in space" ] },

"r0c0" represents row 0 column 0, the top-right most cell. "type": "text" means that the cell can be edited as a "text" based input field.

"r1c1" on the other hand is the cell on the second column, on the second row. Its "type" is an array of values, representing an enum. This meeans that the editable values of a cell will be provided from a dropdown

Double click on a cell to edit it!

The possible types of a cell are

"text", "number", "email" and [...customValues] (up to 3)

For "number" types you can also provide "min" and "max" values

"r0c3": { "type": "number", "min": 17, "max": 100 }
